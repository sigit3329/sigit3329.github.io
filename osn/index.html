Pengantar untuk Siswa

Selamat datang di dunia OSN Informatika! Banyak yang mengira OSN Informatika adalah lomba "mengetik kode paling cepat" atau "menghafal bahasa pemrograman". Itu salah besar.

OSN Informatika adalah lomba berpikir, menyelesaikan masalah (problem solving), dan menemukan pola. Komputer dan bahasa pemrograman hanyalah "alat" yang kita suruh untuk menghitung apa yang sudah kita pikirkan.

Modul ini melatih pola pikir sistematis. Siswa akan belajar konsep gerbang logika (AND, OR, NOT) dan cara mengeliminasi kemungkinan, yang merupakan fondasi sebelum menulis kode apapun.

1. Dasar Logika Berpikir

Tujuan: Melatih pola pikir sistematis dan analitis. Ini adalah fondasi utama!
🔍

Analogi Sederhana: Menjadi Detektif

Seorang detektif tidak menebak-nebak siapa pelakunya, ia menggunakan petunjuk, menyusun fakta, dan mencoret orang yang tidak mungkin menjadi pelaku sampai tersisa satu kebenaran.

A. Logika Dasar (AND, OR, NOT)

Di dalam komputer, segalanya bernilai Benar (True/1) atau Salah (False/0).

  • AND
    Syarat ketat. Analogi: "Kamu boleh naik wahana JIKA tinggi badan > 150 cm AND punya tiket." Jika salah satu saja tidak terpenuhi, kamu tidak boleh naik.
  • OR
    Syarat longgar. Analogi: "Kamu dapat diskon JIKA punya kartu member OR sedang ulang tahun." Punya salah satu saja sudah cukup.
  • NOT
    Membalik keadaan. Analogi: Saklar lampu. NOT Nyala = Mati. NOT Benar = Salah.

➞

A OR B FALSE

B. Implikasi dan Negasi

Implikasi (Jika P maka Q)

Analogi: "JIKA hari hujan, MAKA lantai basah."

Catatan OSN: Jika lantai basah, apakah PASTI karena hujan? Belum tentu! Jangan terbalik logikanya.

Negasi (Bantahan)

Janji: "Jika kamu juara, aku belikan es krim".

Bantahannya:

"Kamu juara, TETAPI aku tidak membelikan es krim." (P AND NOT Q).

C. Penalaran Benar/Salah & Eliminasi

Soal OSN sering berbentuk: "Si A bilang B bohong, B bilang C bohong".

Cara mengerjakannya: Jangan dipikir sekaligus! Buatlah asumsi sementara. "Misalkan A jujur, apa akibatnya? Oh, ternyata bertentangan. Berarti asumsi salah. Coret asumsi A jujur. Lanjut ke kemungkinan lain." Ini disebut penalaran deduktif.

Di sini siswa dikenalkan pada pemrograman sebagai alat. Fokusnya bukan pada hafalan sintaks, melainkan pada konsep variabel, operator matematika dasar (termasuk modulo), percabangan, dan perulangan.

2. Dasar Pemrograman

Tujuan: Memahami cara menyuruh komputer menyelesaikan masalah.
🤖

Analogi Sederhana: Robot Koki

Menulis kode seperti menulis resep masakan. Beri tahu bahan-bahannya (Variabel), langkah-langkahnya (Proses), dan hasil masakannya (Output).

A. Variabel & Tipe Data

Variabel adalah "kotak" penyimpan. Tipe data adalah "label" kotaknya.

  • int : Bilangan bulat (1, -10)
  • float : Desimal (3.14)
  • string : Teks ("Halo")
  • boolean : True/False

B. Operator Dasar

+ - * / adalah operasi biasa.

Hati-hati dengan Modulo (%)

Sisa bagi. Contoh: 10 % 3 = 1 (10 dibagi 3 hasil 9, sisa 1). Sangat sering keluar di OSN!

C. Percabangan (If-Else)

Analogi persimpangan jalan: JIKA lampu merah, MAKA berhenti. SELAIN ITU (hijau), jalan.

if (kondisi) {
  lakukan_ini();
} else if (kondisi_lain) {
  lakukan_itu();
} else {
  lakukan_sisanya();
}

D. Perulangan (For, While)

Analogi: Dihukum menulis 100 kali. Jangan tulis 100 baris, gunakan perulangan.

For Loop

Dipakai jika TAHU berapa kali akan mengulang (misal: pasti 100 kali).

While Loop

Dipakai jika TIDAK TAHU berapa kali, tapi tahu kapan berhenti (misal: lari sampai finish).

Siswa diajak untuk memodelkan proses yang terjadi dalam soal ke dalam "state" atau keadaan tertentu. Disertai simulasi grid 2D untuk memvisualisasikan bagaimana perubahan state direkam langkah demi langkah.

4. Pattern Recognition (Pengenalan Pola)

Tujuan: Menemukan rahasia di balik data jutaan.
📅

Analogi: Prediksi Hari

Jika sekarang Senin, hari apa 1000 hari lagi? Pakai Modulo! 1000 dibagi 7 sisa 6. Hitung 6 hari dari Senin = Minggu. Tidak perlu menghitung manual 1000 kali.

A. Barisan dan Pola Pertumbuhan

Geometri

B. Generalisasi & Segitiga Pascal

Generalisasi artinya membuat rumus umum.
Misal: 1 + 2 + 3 + ... + 100

Daripada dilooping 100 kali, pakai pola Gauss:
(Ujung Kiri + Ujung Kanan) x (Setengah Jumlah Angka)
(1 + 100) x 50 = 5050

Segitiga Pascal sangat penting di OSN karena berhubungan langsung dengan hitungan kombinasi (Kombinatorik).

6. Greedy (Strategi)

Tujuan: Mengambil langkah terbaik saat ini juga tanpa pusing memikirkan masa depan.
💵

Analogi: Memberi Kembalian

Diminta kembalian Rp 17.000 dengan koin/kertas sesedikit mungkin? Secara natural kamu ambil terbesar dulu: Rp 10.000, lalu Rp 5.000, lalu Rp 2.000. Itulah Greedy!

Simulator Kasir Greedy

Pecahan tersedia: 10k, 5k, 2k, 1k, 500

Kapan Greedy Gagal? (Jebakan OSN)

Misal pecahan yang tersedia adalah: 12k, 10k, 2k. Target: 14k.

  • Pakai Greedy: Ambil 12k. Sisa 2k. Ambil 2k. Total butuh 2 koin. (Optimal!)

TAPI, misal targetnya 15k (dengan koin 12k, 10k, 5k):

  • Pakai Greedy: Ambil 12k, sisa 3k. BUNTU! (Tidak ada koin 3k).
  • Padahal bisa: 10k + 5k (2 koin).

Mengenali apakah soal bisa pakai Greedy atau butuh algoritma lebih kompleks (Dynamic Programming) adalah skill tingkat dewa di OSN!

7. Graph Dasar

Tujuan: Menyelesaikan soal tentang jaringan, rute, dan peta.
🗺

Analogi: Peta Angkot / Jaringan Teman

Graph adalah kumpulan kota (Node) yang dihubungkan oleh jalan/rute angkot (Edge).

A. Node dan Edge

  • Node (Titik): Kotanya / Orangnya.
  • Edge (Garis): Hubungannya. Bisa searah (jalan 1 arah), dua arah, atau punya bobot jarak (panjang aspal).

B. Cara Menelusuri Peta (BFS vs DFS)

BFS (Melebar)

Analogi: Jatuhkan batu ke kolam, riak menyebar rata. Cek jarak 1 dulu, baru jarak 2.

Fungsi: Mencari jalan terpendek.

DFS (Mendalam)

Analogi: Masuk labirin. Jalan terus ke satu arah sampai buntu, mundur 1 langkah, belok.

Fungsi: Mengecek konektivitas/terhubung.

C. Shortest Path & MST

  • Shortest Path: Algoritma Dijkstra. Mencari rute A ke B tercepat ala Google Maps.
  • Minimum Spanning Tree (MST): Analogi Presiden aspal jalan. Hubungkan semua kota dengan biaya semurah mungkin. Tidak boleh ada jalan memutar (loop).

9. Brute Force vs Efisiensi

Tujuan: Mengetahui batas kemampuan komputer.
🔒

Analogi: Membuka Gembok

Brute Force: Coba semua kunci di dunia satu per satu.
Efisiensi: Lihat merk gemboknya, lalu ke toko spesifik.

Kapan Pakai Brute Force?

Jika batas data (Constraint / N) kecil, misal N < 10. Silakan mencoba semua kemungkinan. Aman dan pasti benar.

Kapan Butuh Efisiensi?

Jika N = 100.000. Juri cuma kasih 1 detik. Harus pakai rumus Matematika atau Binary Search (Kompleksitas Waktu / Big-O).

Visualisasi: Waktu Eksekusi saat N Membesar

10. Problem Solving OSN

Tujuan: Menggabungkan semua kekuatan ke simulasi perang.

Langkah Menganalisis Soal

1
Baca Soalnya, Buang Dongengnya

Pembuat soal suka bercerita soal Pak Dengklek dan bebeknya. Abstraksikan! Terjemahkan jadi bentuk "Graph", "Array", atau "Matematika".

2
Lihat Batasannya (Constraints)

N kecil = Brute force. N sedang = Simulasi. N sangat besar (Satu Miliar) = Pasti rumus pola/matematika O(1) atau O(log N).

3
Temukan Edge Cases (Kasus Ekstrem)

Apa terjadi jika N=0? Jika jumlah negatif? Program bagus tahan banting terhadap Edge Cases.
